The Obscuraworks Engineering Blog is where our team publishes technical writing about the systems, decisions, and tradeoffs behind the Obscuraworks API Platform.

We write about API infrastructure, media processing, automation architecture, and the day-to-day engineering work of building and operating distributed systems at scale. When we learn something interesting — or make a mistake worth documenting — we write about it here.

## What We Build

[Obscuraworks](https://obscuraworks.org) is a developer-first API platform for teams that need reliable infrastructure for automation, media processing, and API orchestration. Our platform handles request routing and transformation, media encoding and delivery, and durable workflow execution.

## Who Writes Here

Posts are written by members of the Obscuraworks engineering team. Our contributors include backend engineers, infrastructure specialists, and platform architects. Author names and roles are included with each post.

## Topics We Cover

- API design and gateway architecture
- Media processing pipelines (video, image, audio)
- Kubernetes and distributed systems operations
- Developer tooling and CLI design
- Reliability engineering and observability
- Platform performance and optimization
- Announcements and product updates
